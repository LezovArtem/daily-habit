# daily-habit

# Техническое задание

## Daily Habit Tracker

Приложение предназначено для отслеживания ежедневных и еженедельных привычек. Пользователь может создавать привычки, отмечать их выполнение, просматривать статистику и историю активности. Приложение работает как одностраничный интерфейс без backend-части, а данные сохраняются в `localStorage`.

## 1. Структура приложения

### Главная страница

Главная страница содержит:

- название приложения и краткое описание;
- общую статистику по привычкам;
- форму добавления и редактирования привычки;
- фильтрацию привычек по категории;
- список привычек в виде карточек.

Карточка привычки отображает:

- название привычки;
- категорию;
- тип привычки: ежедневная или еженедельная;
- заметки;
- текущую серию выполнения;
- процент успешности;
- прогресс выполнения;
- кнопки: отметить выполнение, редактировать, удалить, открыть детали.

### Страница деталей

Страница деталей содержит:

- название привычки и категорию;
- заметки к привычке;
- кнопку отметки выполнения текущего дня или недели;
- статистику привычки;
- дату последнего выполнения;
- список последних отметок;
- визуальную историю активности в виде heatmap;
- кнопку возврата на главную страницу.

### Основные компоненты

- `App` — корневой компонент приложения.
- `AppHeader` — верхняя часть приложения с заголовком.
- `AppLoader` — индикатор загрузки.
- `HomeView` — главная страница со списком привычек.
- `HabitDetailView` — страница подробного просмотра привычки.
- `HabitForm` — форма добавления и редактирования привычки.
- `HabitCard` — карточка одной привычки.
- `ProgressHeatmap` — визуальная история выполнения.
- `useHabits` — модуль для работы с привычками, статистикой и `localStorage`.

## 2. Технологии

- Frontend-фреймворк: `Vue 3`.
- Тип приложения: `SPA`.
- Сборка проекта: `Vite`.
- Маршрутизация: `Vue Router`.
- Хранение данных: `localStorage`.
- Стилизация: глобальный `CSS`.
- Backend: отсутствует.

Используемые библиотеки:

- `vue`;
- `vue-router`.

## 3. Структура данных и примеры JSON

Основной объект привычки содержит:

- `id` — уникальный идентификатор привычки;
- `name` — название привычки;
- `category` — категория привычки;
- `frequency` — тип привычки: `daily` или `weekly`;
- `createdAt` — дата создания;
- `completions` — массив дат выполнения;
- `notes` — заметка пользователя.

Пример объекта привычки:

```json
{
  "id": 1716200000000,
  "name": "Пить воду",
  "category": "Здоровье",
  "frequency": "daily",
  "createdAt": "2026-05-25",
  "completions": [
    "2026-05-25"
  ],
  "notes": "Выпивать минимум 1.5 литра воды в день"
}
```

Пример хранения данных в `localStorage`:

```json
[
  {
    "id": 1716200000000,
    "name": "Пить воду",
    "category": "Здоровье",
    "frequency": "daily",
    "createdAt": "2026-05-25",
    "completions": ["2026-05-25"],
    "notes": "Выпивать минимум 1.5 литра воды в день"
  },
  {
    "id": 1716200500000,
    "name": "Тренировка",
    "category": "Спорт",
    "frequency": "weekly",
    "createdAt": "2026-05-25",
    "completions": ["2026-05-25"],
    "notes": "Минимум одна тренировка в неделю"
  }
]
```

Дополнительно в интерфейсе рассчитываются:

- выполнена ли привычка сегодня;
- выполнена ли привычка на текущей неделе;
- общее количество выполнений;
- текущая и максимальная серия;
- процент успешности;
- дата последнего выполнения.

Ключ хранилища:

```text
daily-habit-tracker:v1
```

## 4. План реализации требований

### Обязательные требования

1. Загрузка приложения

- При запуске отображается loader.
- Данные загружаются из `localStorage`.
- Если данных нет, отображается пустое состояние.

2. Добавление привычки

- Пользователь вводит название, категорию, тип привычки и заметку.
- Нельзя создать привычку с пустым названием.
- После сохранения привычка добавляется в список и записывается в `localStorage`.

3. Просмотр списка привычек

- Все привычки отображаются на главной странице в виде карточек.
- В карточке видны основные данные и прогресс.
- Выполненные привычки визуально отличаются от невыполненных.

4. Редактирование и удаление

- Пользователь может изменить данные привычки через форму.
- При редактировании сохраняются `id`, дата создания и история выполнений.
- Пользователь может удалить привычку из списка.

5. Отметка выполнения

- Ежедневную привычку можно отметить один раз в день.
- Еженедельную привычку можно отметить один раз в неделю.
- Повторное нажатие снимает отметку.
- После изменения данные сразу обновляются на главной странице и в деталях.

6. Навигация

- Главная страница: `/`.
- Страница деталей: `/habit/:id`.
- На странице деталей есть кнопка возврата назад.

7. Сохранение данных

- Все изменения автоматически сохраняются в `localStorage`.
- После перезагрузки страницы данные восстанавливаются.

8. Фильтрация

- Пользователь может фильтровать привычки по категории.
- Также доступен вариант отображения всех привычек.

9. Интерфейс

- Интерфейс должен быть простым и понятным.
- Приложение должно корректно работать на компьютере и телефоне.
- В консоли браузера не должно быть ошибок.

### Дополнительные требования

1. Поддержка типов привычек

- Приложение поддерживает ежедневные привычки.
- Приложение поддерживает еженедельные привычки.

2. Статистика

- Для каждой привычки рассчитывается текущая серия.
- Отображается максимальная серия.
- Рассчитывается процент успешности.
- Показывается дата последнего выполнения.

3. История активности

- Последние выполнения отображаются списком.
- История показывается в виде heatmap.
- Для ежедневных и еженедельных привычек используется разный период отображения.

4. Категории

- Пользователь может указать категорию привычки.
- Категории собираются автоматически на основе созданных привычек.
- Список привычек можно фильтровать по категории.

5. Работа без backend

- Приложение не требует сервера.
- Все данные хранятся локально.
- Для работы не нужна регистрация пользователя.
